# CaseMind Defense — מדריך משתמש

מערכת ניהול וניתוח ראיות לעורכי דין וחוקרים. הכול רץ מקומית על המחשב
שלך — שום חומר לא עוזב את המכונה.

**שפות נתמכות:** עברית, רוסית ואנגלית — כולל מסמכים סרוקים (OCR),
זיהוי ישויות, וחיפוש חוצה-שפות (שאילתה בעברית מוצאת מסמך ברוסית).

## התקנה (פעם אחת)

1. פתח PowerShell בתיקיית הפרויקט והרץ:
   ```powershell
   powershell -ExecutionPolicy Bypass -File scripts\setup.ps1
   ```
2. בסיום יופיע קיצור דרך **CaseMind Defense** על שולחן העבודה.
3. דאבל-קליק — האפליקציה מפעילה את השרת לבד ועולה תוך שניות.

## עבודה יומיומית

### 1. תיק חדש וייבוא ראיות (דף Evidence)
- **New Case** ⟵ שם התיק ⟵ בחר אותו ברשימת ה-Case.
- **Import Evidence** ⟵ בחר קובץ (PDF, סרוק או דיגיטלי · תמונות · טקסט).
- הקובץ נקלט מיד בסטטוס `processing`; העיבוד (OCR עברית, אינדוקס,
  זיהוי ישויות) רץ ברקע — ‏**Refresh** יראה `indexed` כשמוכן.
- קובץ שכבר קיים ייחסם עם הודעה (זיהוי כפילויות SHA256 — שרשרת משמורת).
- לחיצה על שורה: ‏Preview במרכז, מטא-דאטה מלאה מימין.

### 2. חיפוש (דף Search)
- ‏**Semantic** — חיפוש לפי משמעות; עובד גם חוצה-שפות (שאילתה בעברית
  תמצא מסמך באנגלית).
- ‏**Keyword** — התאמה מדויקת של ביטוי.
- **דאבל-קליק על תוצאה** קופץ לראיה עם הקטע מודגש.

### 3. שאלות לתיק (דף AI Workspace)
שאל בעברית חופשית ("מה טוען הנאשם לגבי...?"). התשובה מבוססת אך ורק על
הראיות, עם ציטוטים [1][2] — דאבל-קליק על ציטוט פותח את המקור.
בלי Ollama מותקן תקבל את הציטוטים בלבד (המערכת מציינת באיזה מצב ענתה).

### 4. תמונת החקירה
- ‏**Timeline** — כל התאריכים שנמצאו בראיות, ממוינים; דאבל-קליק למקור.
- ‏**Entities** — אנשים, מקומות, טלפונים, ת"ז, לוחיות; דאבל-קליק מחפש מופעים.
- ‏**Entity Graph** — מי קשור למי דרך אילו ראיות (גודל = אזכורים).
- ‏**Contradictions** — זוגות קטעים דומים שה-AI סימן כסותרים, עם הסבר.
  הטעינה הראשונה אורכת עד כ-2 דקות (שיפוט AI).

### 4ב. הקלטות ווידאו (האזנות, חקירות מצולמות)
ייבא `wav / mp3 / m4a / mp4 / mov` כמו כל ראיה — המערכת מתמללת ברקע
(סטטוס `transcribed` בסיום) והתמליל נכנס לחיפוש, לישויות ול-AI.
**הציטוטים הם חותמות זמן** (`time:14:32-15:10`) — מוצאים את הדקה
המדויקת בהקלטה של שעתיים. תמלול איטי מהאינדוקס הרגיל (עד ~פי 2 מאורך
ההקלטה על CPU) — רץ ברקע, אפשר להמשיך לעבוד.

### 5. דוח לתיק (כפתור Report בדף Evidence)
מפיק דוח HTML בעברית: מצאי ראיות עם SHA256 (שרשרת משמורת), ציר זמן,
ישויות ויומן פעולות. נפתח בדפדפן — ‏**Ctrl+P** להפקת PDF.

### 6. תחזוקה (דף Settings)
- ‏**Verify Evidence Integrity** — מוודא שאף קובץ ראיה לא שונה מאז קליטתו.
- ‏**Create Backup** — גיבוי מלא (DB + כל הראיות) לקובץ zip אחד תחת
  `backend\data\backups`. שחזור: פתח את ה-zip והחזר את הקבצים למקום.

## הגדרות מתקדמות (משתני סביבה)

| משתנה | ברירת מחדל | תפקיד |
|--------|-------------|--------|
| `CASEMIND_API_KEY` | כבוי | דרישת מפתח לכל קריאת API (להגדיר בשני הצדדים) |
| `CASEMIND_IMPORT_ROOTS` | הכול | הגבלת ייבוא לתיקיות מסוימות |
| `CASEMIND_LLM_MODEL` | qwen2.5:3b-instruct | מודל ה-AI. **מומלץ: ‏`aya-expanse:8b`** — עברית מצוינת (במבחן ראש-בראש: תשובה משפטית מובנית מול ג'יבריש); על GPU של 4GB ייקח ~דקה לתשובה, על 12GB+ שניות בודדות |
| `CASEMIND_PDF_OCR_MAX_PAGES` | 50 | תקרת עמודי OCR לקובץ סרוק |
| `CASEMIND_WHISPER_MODEL` | small | מודל תמלול. **מומלץ לעברית: ‏`ivrit-ai/faster-whisper-v2-d4`** (במבחן: תמלל נכון את מילות המפתח שהמודל הרגיל שיבש); להקלטות שרובן רוסית — ‏`small`/`medium` |

## פתרון תקלות

- **האפליקציה לא עולה / "Backend Unavailable"** — הרץ שוב את
  `scripts\setup.ps1`; ודא ש-`backend\.venv` קיים.
- **‏OCR עברית/רוסית לא עובד** — המערכת משתמשת בחבילות שפה מקומיות תחת
  `backend\data\tessdata` (עברית, רוסית ואנגלית מותקנות אוטומטית); ניתן
  לכפות שפות עם `CASEMIND_OCR_LANGS` (למשל `eng+heb+rus`).
- **ה-AI עונה "citations only"** — ‏Ollama לא רץ; הפעל `ollama serve`
  או התקן דרך setup.ps1.
- **לוגים** — ‏`backend\data\logs\backend.log`; צרף אותו לכל דיווח תקלה.
